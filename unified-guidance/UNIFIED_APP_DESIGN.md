# 통합 가이던스 앱 — 아키텍처 설계

> 한 태블릿·한 앱에서 **자율조향 + RTK 균평 동시 제어**.
> 기존 자율주행 키트(AGMO/CHCNAV/FJDynamics) 보유 고객에게
> **안드로이드 앱 + RTK 하드웨어**를 판매하는 비즈니스 모델 전용 설계.

---

## 1. 시장 문제 (왜 이게 차별점인가)

| 경쟁사 | 자율조향 | 균평 | 동시 사용 |
|--------|---------|------|----------|
| CHCNAV | ✅ AgNav | 별도 | ❌ 태블릿 2대 또는 택1 |
| FJDynamics | ✅ | 별도 | ❌ 태블릿 2대 또는 택1 |
| AGMO | ✅ | ✕ | — |
| **본 제품** | ✅ | ✅ | **✅ 한 태블릿, 한 앱** |

CHCNAV·FJD는 자율조향과 균평을 한 화면에서 못 돌린다. 태블릿을 2대 달거나,
한 대면 둘 중 하나만 쓴다. **본 앱은 단일 태블릿에서 둘 다 구동**한다.

---

## 2. 핵심 설계 원리 — 단일 RTK 백본, 이중 제어

```
                  ┌─────────────────────────────────────────┐
                  │          통합 앱 (안드로이드)              │
                  │                                          │
   F9P (USB) ───► │  ┌────────────────────────────────────┐ │
   LoRa (USB) ──► │  │   공유 측위 백본 (Positioning Core)  │ │
                  │  │   NMEA 파싱 · NTRIP/RTCM · ENU 변환  │ │
                  │  │   상태추정 (위치/자세/높이)           │ │
                  │  └───────────┬─────────────┬───────────┘ │
                  │              │             │             │
                  │   ┌──────────▼──┐   ┌──────▼──────────┐  │
                  │   │ 자율조향 모듈 │   │  RTK 균평 모듈   │  │
                  │   │ autosteer_   │   │  leveler_       │  │
                  │   │ core         │   │  core           │  │
                  │   └──────┬──────┘   └──────┬──────────┘  │
                  └──────────┼─────────────────┼─────────────┘
                             │ CAN             │ CAN
                  ┌──────────▼──┐   ┌──────────▼──────────┐
                  │  조향 모터   │   │  STM32 → Kubota(7)  │
                  │  + 앵글센서  │   │  레이저 커넥터       │
                  └─────────────┘   └─────────────────────┘
                   (기존 키트 부품)    (본 제품 추가 하드웨어)
```

**핵심: GNSS 안테나(F9P) 하나가 조향과 균평을 동시에 먹인다.**
- 조향은 평면 위치(x, y)와 방위가 필요 → 백본이 제공
- 균평은 높이(z)가 필요 → 같은 F9P가 동시에 제공
- NTRIP 보정도 하나로 둘 다 cm급 정밀도

경쟁사가 태블릿 2대를 쓰는 이유는 각 앱이 자기 GNSS·자기 화면을 점유하기 때문.
본 설계는 **측위를 공유**하고 **출력만 둘로 분기**하므로 한 태블릿으로 충분하다.

## 3. 벤더 중립 — AGMO/CHC/FJD 하드웨어에서 다 돈다

본 앱은 기존 자율주행 앱과 **직접 통신하지 않는다.** 의존성 0.

### 3.1 측위: 자체 F9P (벤더 GNSS 불needed)
기존 키트의 GNSS에 접근할 필요 없음. 본 제품이 제공하는 F9P를 USB로 직접 읽어
독립적으로 cm급 위치·높이를 산출. → 어떤 벤더든 무관.

### 3.2 조향 출력: 두 가지 호환 모드
고객 상황에 맞춰 선택:

**모드 A — 균평 전용 (가장 안전, 기본 권장)**
- 기존 벤더 자율조향을 그대로 사용 (CHCNAV가 조향, 운전자가 신뢰하는 그대로)
- 본 앱은 **균평만** 담당
- 단일 태블릿 문제 해결: 본 앱이 벤더 앱 위 오버레이로 떠서 균평 UI 제공
- 조향 모듈은 표시만(읽기) 또는 비활성

**모드 B — 통합 조향 (본 제품이 조향까지)**
- 본 앱의 autosteer_core가 기존 조향 모터를 CAN으로 직접 구동
- 벤더 앱 불필요, 한 앱이 조향+균평 완전 통합
- 전제: 해당 키트의 모터 CAN 프로토콜 확보 (AGMO 경유 등)

→ UI·백본은 동일. 조향 모듈의 **출력단만** 모드에 따라 분기.
   고객이 "조향은 CHC 그대로 쓰고 균평만 추가" 하면 모드 A,
   "한 앱으로 다 통합" 하면 모드 B.

### 3.3 균평 출력: Kubota (7) 레이저 커넥터 (벤더 무관)
균평은 트랙터 자체 인터페이스(레이저 커넥터)를 쓰므로 자율주행 벤더와 완전 독립.

## 4. UI 설계 — 두 기능 동시 사용 (unified_ui.html)

트랙터 캐빈 = 진동·햇빛·장갑·곁눈질 환경. 산업용 계기판 원칙:

- **두 라이브 오차를 항상 동시 표시** (탭 전환 없음) — 이게 핵심
  - 조향: 크로스트랙 오차(XTE) 대형 숫자 + 좌우 바
  - 균평: 그레이드 오차 대형 숫자 + 깎기/메우기 바
- **색으로 모듈 구분**: 조향=청록, 균평=황금 → 곁눈질로 즉시 식별
- **상태색**: 녹색=정상추종/온그레이드, 황색=보정중, 적색=정지/이상
- **독립 engage**: 조향만/균평만/둘다 자유 조합 (모드 A는 균평만 켬)
- **통합 맵**: 주행 라인 + 균평 완료영역 + 깎기/메우기를 한 화면에
- **큰 탭 영역**: 목표높이 ▲▼, HOLD, 프로파일(일반/과부하/모래), 비상정지
- **비상정지**: 둘 다 즉시 정지

레이아웃 (10인치 가로):
```
┌── 상태바: RTK·위성·NTRIP·LoRa·속도·정밀도 (공유) ──────────┐
├────────────────────────┬───────────────────────────────────┤
│                        │  자율조향          [조향 ON]        │
│      포장 맵            │   XTE  −2.1 cm   ● 추종중           │
│   라인 + 완료영역       │   ◄───●───►   조향각 4.2°           │
│   + 깎기/메우기         │   [일반][과부하][모래]              │
│                        ├───────────────────────────────────┤
│                        │  RTK 균평          [자동 ON]        │
│                        │   GRADE +0.4 cm  ● 온그레이드       │
│                        │   ▼ 목표 13.0 cm ▲   [HOLD]         │
├────────────────────────┴───────────────────────────────────┤
│ [■ 비상정지]  단일 RTK 백본이 조향·균평 동시 구동  [≡ 설정]   │
└──────────────────────────────────────────────────────────────┘
```

## 5. 소프트웨어 구성

```
unified-guidance/
├── core/
│   ├── positioning_core.py     # 공유: NMEA·NTRIP·ENU·상태추정 (신규, 두 코어에서 분리)
│   ├── autosteer_core.py       # 기존 (조향 4계층)
│   └── leveler_core.py         # 기존 (균평 4계층)
├── android/                    # 안드로이드 앱
│   ├── ui/                     # unified_ui 구현 (Compose)
│   ├── io/                     # F9pUsbClient, LoraNtripUsbClient, CAN
│   └── modes/                  # 모드 A(균평전용) / 모드 B(통합조향)
└── ui-prototype/
    └── unified_ui.html         # UI 프로토타입 (이 산출물)
```

**다음 작업:**
1. positioning_core 분리 — 두 코어의 RTK/NTRIP/ENU 중복을 공유 백본으로 추출
2. 모드 A/B 스위치 구현 (조향 출력단 분기)
3. unified_ui.html → Android Compose 이식
4. 단일 태블릿 통합 테스트 (F9P 1대 → 조향+균평 동시)

## 6. 비즈니스 정합성

- 판매 대상: AGMO/CHC/FJD 키트 보유 고객 (조향은 이미 있음)
- 판매 품목: 안드로이드 앱 + RTK 하드웨어(F9P, LoRa, STM32, 케이블)
- 가치 제안: "기존 자율조향에 **균평을 추가**, 게다가 **한 태블릿에서 동시에**"
- 모드 A로 진입장벽 최소화(조향 안 건드림) → 신뢰 확보 후 모드 B 업셀 가능
