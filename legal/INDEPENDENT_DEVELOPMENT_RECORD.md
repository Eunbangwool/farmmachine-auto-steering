## 1. 개발 원칙 선언

본 프로젝트는 다음 원칙을 준수하여 개발되었습니다.

1. **호환성 목적**: 이용자가 정당하게 소유한 농기계 하드웨어와의 호환성
   확보를 목적으로 합니다. 한국 저작권법 제101조의4(프로그램코드 역분석)가
   허용하는 호환 목적 범위 내입니다.

2. **코드 비복제**: 제3자 소프트웨어의 소스코드를 복사하지 않았습니다.
   분석을 통해 얻은 것은 **인터페이스 사양(통신 프로토콜, CAN 메시지
   포맷)**이며, 이는 아이디어·사실·기능의 영역으로 저작권 보호 대상이
   아닙니다.

3. **독립 재작성**: 모든 알고리즘과 코드는 공개된 학술 자료·공개 문서·
   제조사 공식 매뉴얼을 근거로 자체적으로 재작성했습니다.

## 3. 개발 방법론 (Clean Room 유사 접근)

본 프로젝트는 다음 절차로 개발되었습니다.

1. **사양 추출 단계**: 공개 매뉴얼과 인터페이스 분석으로 "무엇을 해야
   하는가"(통신 프로토콜, 명령 포맷)를 사양으로 정리했습니다. 이 단계에서는
   기능 사양만 기록하고 제3자 코드 표현은 기록하지 않았습니다.

2. **독립 구현 단계**: 위 사양을 바탕으로 공개 알고리즘을 사용해 코드를
   처음부터 작성했습니다. 사용 언어·구조·변수명·주석 모두 자체적으로
   결정했습니다.

3. **검증 단계**: 구현이 사양과 일치하는지 시뮬레이션으로 검증했습니다.
   (예: KY170C 인코딩 함수가 Keya 매뉴얼 예제 7건과 일치함을 확인)

## 4. 코드 독립성 증거

### 4-1. 우리 코드의 고유 특징

```
- 언어: Python (autosteer_core.py) + Kotlin (Android)
- 구조: 4계층 추상화 (자체 설계)
- 알고리즘 선택: Pure Pursuit + ImplementReferenced (공개 기법)
- UI: Jetpack Compose 자체 작성 (AGMO는 Qt6 C++ → 구조적으로 상이)
- 변수명·주석: 한국어 자체 작성
```

### 4-2. 제3자 코드와의 비유사성

```
- AGMO는 Qt6 C++ / 우리는 Kotlin Compose → 언어·프레임워크 상이
- AGMO의 .so는 컴파일된 바이너리 / 우리는 소스부터 자체 작성
- 우리는 .so의 함수를 "호출"할 뿐, 내부 구현을 복제하지 않음
```
